How to make an image suitable for Modelr

Here are a few tips to keep in mind so that an image can be effortlessly read into an earth model. If you can't easily re-work an illustration to fit the requirements of a clean earth model, it might be easier to redraw the model using a graphics program such as Corel Draw, Adobe Illustrator, or Inkscape.

Remove annotation, labels, and rock outlines

Each color is mapped to a rock class, so in order to avoid annotations and labels from being turned into rock classes, you need to remove them from the image.

Remove annotations

Remove anything that you don't want in the whitespace. This includes text, text labels, scales, legends, and so on..

Remove borders, grid lines, and axis tick marks

Since Modelr is able to convert earth models between the depth and travel-time domains, it is important remove any distinct borders around the edge of the image, otherwise their might be non-physical reflections along the border..

Remove rock outlines

This is perhaps the most critical element to consider. If outlines and borders between rock are present they will be mapped to their own rock class and negate the seismic reflectivity contrasts that you are after..

So only rocks remain

Save the file as a PNG or JPEG, preferably with no alpha channel or dithering.

Use color wisely

Color sensitivity

Colors should be distinct, but they don't have to be too distinct. The level of contrast in this example will work just fine. Modelr classifies rocks based on unique R,G,B color values. If you want to make and earth model with 5 rocks in it, the image should ideally only have 5 colors in it. The colors are be arbitrary.

Color assignment

If the picture has more colors than you need, you will be able to set two different colors to the same rock class. You cannot however set two different rocks to the same color, even if the color exists in more than one place.

Geological information density

The finer details are fine, ...

Models can contain a high level of detail, resolution and complexity, as long as all the colors involved each comprise more than 1% of the total area of the image. We've designed the image classification system this way for practical purposes. The result is that you'll never need to use more than 99 rocks, because you will never have more than 99 colors taking up greater than 1% of the image.

... but small bits of color will be skipped

In order to deal with dithering, blended colors, and other issues with digital images, Modelr will not create a rock classification for a particular color instance for each color that takes up less than 1% of the total image area.

Crop the image and zoom in

If a rock target is not being identified, it could be because it is too small. you may be imposing model dimensions that are larger than they need to be. Consider cropping the image and focusing into the area you are interested in.

Regarding resolution, size, and scales

Avoid visible pixels

If you can see the pixels, you don't have enough resolution. You'll get excessive edge effects when you run your seismic experiment. It better to have an image that is slightly higher resolution that you'd use for illustrations and graphics.

Minimum image dimensions

By default, the number of traces in the synthetic will be the same as the width of the image in pixels. As a result, one way to add more traces to your synthetic model, is to boost the resolution of your input image. Beyond that, there are no other constraints on image dimensions, but we recommend keeping files smaller than 1 MB for speed.

The vertical scale can be changed

The vertical scale is something that is set after the image has been uploaded, so the aspect ratio can be essentially anything that allows you to achieve the information density and geological complexity that you want.
