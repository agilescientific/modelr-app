Making an image for Modelr

Here are a few tips to keep in mind so that an image can be effortlessly read into an earth model. If you can't easily re-work an illustration to fit the requirements, it might be easier to redraw the model using a vector graphics program such as Inkscape, which is free, or Adobe Illustrator.

Remove annotation, labels, and rock outlines

Each color is mapped to a rock class, so in order to avoid annotations and labels from being turned into rock classes, you need to remove them from the image.

Remove annotations

Remove anything that you don't want to be detected and classified as a rock. This includes text, text labels, scales, legends, and so on..

Remove borders, grid lines, and axis tick marks

Since modelr is able to convert earth models between the depth and travel-time domains, it is important remove any distinct borders around the edges of the image, otherwise you'll get non-physical reflections along the border..

Remove rock outlines

This is perhaps the most critical element to remember. If you have outlines between rocks, the color of the outline could be mapped to its own rock class. This would of course be an unnatural situation, and negate the seismic reflectivity contrasts that you are after..

So only rocks remain

Save the file as a PNG or JPEG, preferably with no alpha channel or dithering.

Use color wisely

Color sensitivity

Colors should be distinct, but they don't have to be too distinct. For instance, the contrast levels in the image show to the left will work just fine. Modelr classifies rocks based on unique R,G,B color values. If you want to make an earth model with 5 rocks in it, the image should therefor only have 5 colors in it. The colors you choose are up to you, and are handled arbitrarily.

Color assignment

This may not be suitable in all cases, but if the picture you are trying to upload has more colors within it than you need, note that you can set two or more different colors to the same rock class after you have imported it. You cannot however set two different rocks to the same color (as shown by the image to the left), even if the color is present in more than one region of the image.

Geological information density

The finer details are fine, ...

Models can contain a high level of detail, resolution and complexity, as long as all the colors involved each comprise more than 1% of the total of the image area. We've deliberately designed the image classification interface in this way for practical purposes. The upshot is that you'll never need to use more than 99 rocks, because it is not possible to have more than 99 colors where each one takes up more than 1% of the image.

... but small bits of color will be skipped

In order to deal with dithering, blended colors at interfaces, and other issues with digital images, Modelr will not create a rock classification for any color instance that takes up less than 1% of the total image area.

Crop the image and zoom in

If a color is not being identified by the rock classification system, it could be because it is too small. This could be a symptom that you may be imposing model dimensions that are larger than they need to be. Consider cropping the image to zoom in and focus on the area you are interested in.

Resolution, size, and scales

Avoid visible pixels

If you can see individual pixels, then image probably doesn't have enough resolution and your seismic experiment will suffer from edge effects, It is better to have an image that is slightly higher resolution than what you might normally use for normal illustrations and graphics.

Minimum image dimensions

By default, the number of traces in the synthetic will be the same as the width of the image in pixels. As a result, one way to add more traces to your synthetic model, is to boost the resolution of your input image. Beyond that, there are no other constraints on image dimensions, but we recommend keeping files smaller than 1 MB for speed.

The vertical scale can be changed

The vertical scale is something that is set after the image has been uploaded, so the aspect ratio can be essentially anything that allows you to achieve the information density and geological complexity that you want to model.
